# Release Checklist

Gunakan checklist ini saat mendekati rilis (terutama final).

## 1. Cakupan Konten

- [ ] Struktur `book-1` sampai `book-5` valid.
- [ ] Lesson yang ditarget rilis sudah terisi baseline.
- [ ] Tidak ada file wajib lesson yang kosong total.

## 2. Kualitas Konten

- [ ] Contoh Jepang + romaji + Indonesia tersedia.
- [ ] Grammar punya pola, arti, dan contoh memadai.
- [ ] Tidak ada typo kritis pada kosakata utama.

## 3. Konsistensi

- [ ] Penamaan file/folder konsisten.
- [ ] Referensi ke `materials/shared` benar.
- [ ] Tidak ada link internal putus.

## 4. Logging & Versi

- [ ] `CHANGELOG.md` global diperbarui.
- [ ] `CHANGELOG.md` per book diperbarui.
- [ ] Versi rilis ditentukan sesuai `VERSIONING.md`.
- [ ] Jika perlu, buat release candidate (`-rc.1`) sebelum final.

## 5. Finalisasi

- [ ] Tentukan catatan rilis singkat (apa yang selesai, apa yang belum).
- [ ] Lock versi final (mis. `v1.0.0`).
